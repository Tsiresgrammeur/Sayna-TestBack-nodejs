Comment installer les dépendances

Tapez le commande suivant:
  • git clone https://github.com/Tsiresgrammeur/Sayna-TestBack-nodejs
  • cd Sayna-TestBack-nodejs
  • npm install
  • npm start
Le port qui écoute les requêtes est 8000

Concernant sur l'utilisateur

Pour créer un compte, vous devez spécifier les attributs suivants dans "body" sous postman suivants:
  • firstname
  • lastname
  • email
  • sexe
  • role
  • dateNaissance
  • subscription
Le sexe c'est soit 'M' qui veut dire masculin, soit 'F' qui veut dire féminin.
Le role sur l'utilisateur est soit "ROLE_ADMIN" qui a tous les droits soit "ROLE_USER" qui a des droits limité.
Le format de la date de naissance est 'YYYY-MM-DD'.
les valeurs possibles de subscription sont: "LITE","ACCESS","PREMIUM", manque de liste déroulantes, on ajoute ces valeurs manuellement
Il y des utilisateurs par défaut dans la base de donnée exemple:
email:tsiresy@gmail.com password: passwd
Pour se connecter, il faut donner l'email et le mot de passe via les attributs respectives email et password.
Pour supprimer un utilisateur, il faut spécifier son "id" dans "body" sous postman.

L'abonnement

Concernant sur l'abonnement, il faut entrer l'id du carte et le cvc. Le cvc a une convention de chaine à 3 chiffres . L'abonnement requiert un attribut 'montant' qui a pour valeur de 30 minimum, en supposant que 30 euros est le tarif minimal de l'abonnement. Si vous mettez le valeur 30 au 'montant', l'abonnement sera de type "LITE", 40 pour "ACCESS", 50 pour "PREMIUM".
Une carte à une abonnement vide aura un abonnement PREMIUM de 5 minutes pour son premier abonnement.
Pour tester, il y une carte vide à un id "2AM1joJknI307e3egJn4".

La carte bancaire

L'attribut "cartNumber" doit avoir minimum 8 caractères, l'attribut "month" doit avoir la valeur entre 1 à 12.

Pour le son

Un utilisateur n'a pas accès au son si son type d'abonnement dans l'attribut "subscription" est de "PREMIUM"
